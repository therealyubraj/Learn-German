This session focused on evolving a simple "streak-based" word selection algorithm into a sophisticated Tiered Spaced-Repetition System (SRS).The goal was to balance three competing needs: prioritizing difficult words, ensuring long-term retention (handling staleness), and preventing "New Word Avalanche" (overwhelming the user with too much new content).
1. The Core Data ModelWe moved away from simple correct/incorrect tallies to a state-based structure that tracks progress through levels.mastery: A constant-tier level (1, 2, 3...). It acts as the "braking system"—higher levels make words appear less frequently.successCount: A "buffer" that requires the user to get a word right $N$ times at the current level before it levels up.lastReviewed: A Unix timestamp used to differentiate between the Dormant Pool (0) and the Active Pool (>0).
2. The Weight FormulaThe "Staleness" or "Urgency" of a word is calculated using a Time-to-Mastery ratio:$$Weight = \frac{(\text{HoursSinceReview} + 0.1) \times 100}{\text{Mastery}}$$Impact of Failure: If a user gets a word wrong, mastery resets to 1. Because it is the denominator, the weight spikes, forcing the word back into the quiz immediately.Impact of Mastery: As mastery increases, the weight grows more slowly over time, naturally spacing out reviews.
3. The "Active Pool" & Injection LogicThis is the most critical part of the algorithm. It prevents the user from seeing new words if they are currently struggling with the ones they've already started.Healthy Pool Check: The algorithm looks for the highest weight in the Active Pool (maxPriority).Drip-Feeding: New words are only "injected" from the Dormant Pool if the active pool is "healthy" (meaning no reviews are currently urgent or the pool is empty).Selection Priority:Inject New: If maxPriority < POOL_HEALTHY_SCORE.Weighted Review: If the pool is "unhealthy," pick an active word via weighted random selection (staler/harder words have higher odds).Emergency Fallback: If everything is filtered out (e.g., only 1 word in database), return a random word to prevent a crash.
4. Behavioral CharacteristicsPersistence: By using the lastReviewed timestamp, the algorithm "remembers" which words are in the Active Pool across sessions without needing a separate database table.Cold Start: On a brand new account, the Active Pool is empty, so the algorithm automatically triggers the "Injection" logic to start the first batch of words.The Level-Up Gap: When a word levels up, lastReviewed is updated to now and mastery increases. This creates a "double-whammy" that effectively hides the word from the user for a significant amount of time, allowing it to "cool down."
